### Design Interview (from design.md)
- Explicit caller-set model on Agent tool: RESPECT it; only inject when the model param is absent; optionally emit advisory when router disagrees. Bypasses unaffected.
- Code layout: modular stdlib-only package (router/ with classifier, taxonomy, config, effort, cli_fallback modules) + thin hook entrypoint scripts; tests import real modules.
- CLI/hook timeout budget: architect decides from measurement (measure cold/warm `claude -p --model haiku` latency on this machine, then pick CLI subprocess timeout + hook timeout numbers from data).
- Chosen approach: Modular v2 package -- full rewrite per approach B, packaged as testable stdlib-only modules with thin hook entrypoints.

## Learnings (design phase)
- A-1 resolved: Agent tool `model` accepts aliases AND full IDs per docs, but live tool schema (Claude Code 2.1.212) shows enum ["sonnet","opus","haiku","fable"] only; design emits aliases (safe intersection).
- A-2 resolved: default effort is `high` on the whole v2 ladder (xhigh default only on Opus 4.7, not on ladder); persisted via `effortLevel` settings key which accepts low/medium/high/xhigh but NOT max (max is session-only) -- autoswitch clamps max->xhigh on settings write.
- A-3 resolved: multi-hook permissionDecision precedence documented (deny>defer>ask>allow); updatedInput merge across hooks is UNDOCUMENTED -- design keeps output idempotent and documents the limitation.
- A-4 resolved: hook env inheritance undocumented but child processes practically inherit; CLAUDE_CODE_SUBAGENT_MODEL confirmed to outrank per-invocation model param (incl. hook-injected) -- design does best-effort os.environ check + systemMessage warn.
- A-5 confirmed: only PreToolUse/PermissionRequest carry updatedInput; UserPromptSubmit explicitly cannot replace prompt or set model/effort (fields: decision:block, reason, additionalContext, systemMessage) -- self-managed CLI subprocess required for the LLM fallback.
- A-6 resolved: autoswitch write path = `model` + `effortLevel` keys in ~/.claude/settings.json (both empirically present in local settings); precedence managed > ANTHROPIC_MODEL > settings.local > project > user; settings model change needs restart or /model (new-sessions-only claim confirmed).
- Measured latency (this machine): trivial `claude -p --model haiku` 1.83-2.19s (n=5); realistic classifier prompt 4.23-5.47s (n=4). Chosen: CLI subprocess timeout 8s, UserPromptSubmit/PreToolUse hook timeout 10s, SessionStart 5s. v1's 2s timeout would kill 100% of CLI fallback calls.
- RECURSION HAZARD found: headless `claude -p` runs hooks, so the CLI fallback would re-trigger our own UserPromptSubmit hook. Mitigation designed: CLAUDE_MODEL_ROUTER_CHILD=1 env guard set on the subprocess; all entrypoints exit 0 when it is present.
- Doc drift is worse than research stated: README.md embeds a THIRD variant of the hook script (settings-write autoswitch + "Enter to resend"), differing from both committed code (warn-only) and prompt.md (systemMessage autoswitch).
- prompt.md copies are byte-identical (root vs plugins/...); install.sh copies differ only in path resolution -- design makes root install.sh delegate to the plugin copy.
- Local settings.json had effortLevel=medium (user-set), reinforcing that effort mismatch warnings need a distance>=2 damper to avoid nagging.
- Intentional v1 semantic change decided: haiku session + implementation/debugging now up-routes to sonnet (v1 refused haiku->sonnet, pinned by test-hook.sh:94-96; capability-wrong); test to be updated, change documented in CHANGELOG.
- Review reconciliation (spec-reviewer, consistency): added "Requirements Amendments" section to design.md; requirements.md AC-1.1/AC-1.2 reworded ("match" = same tier AND effort distance < effort_warn_distance; tier mismatch always warns) and AC-4.2 scoped to default taxonomy cells (overridden targets degrade to model-only injection). FR-6 suffix preservation clarified as warn/settings-write only (injected models are alias-only per A-1). Config schema split capability_gates (min-tier) vs new effort_floors key (patterns + floor, data-handling defaults). classify_heuristic pseudocode aligned with decision ladder (margin AND top >= 3). run_eval.py gains NFR-1 p95 < 200ms assertion; xhigh matrix cells got explicit effort-first rationale.
